# 七言绝句生成系统报告

## 1. 任务说明

本项目实现一个基于字符级 GRU 的七言绝句生成系统。输入可以是首句或藏头字，输出为四句七言绝句，每句 7 个汉字。

支持两种条件生成方式：

| 模式 | 输入 | 输出 |
|---|---|---|
| 首句续写 | 7 字首句 | 保留首句，生成后 3 句 |
| 藏头诗 | 4 个藏头字 | 生成完整 4 句，每句首字对应输入 |

采样策略设置为三组：

| 策略 | 参数 |
|---|---|
| stable | temperature = 0.7 |
| balanced | temperature = 1.0 |
| creative | temperature = 1.3 |

## 2. 数据处理

数据来源为课程给定的 ancient-poems-dataset：

```text
https://dicalab-scu.github.io/nlp/post/ancient-poems-dataset/
```

本项目只保留七言绝句子集，筛选条件为：

| 条件 | 说明 |
|---|---|
| 句数 | 4 句 |
| 每句长度 | 7 个汉字 |
| 标点 | 按原始诗句标点处理，训练序列中保留逗号、句号等标点信息 |
| 标签 | 删除情绪、题材等额外标签，只保留诗句文本 |

处理后数据规模：

| 数据项 | 数量 |
|---|---:|
| 严格七绝数量 | 136631 |
| 训练集 | 122967 |
| 验证集 | 6831 |
| 测试集 | 6833 |
| 字符数 | 4993 |

## 3. 模型结构

本项目只使用 GRU，不引入 LSTM 对比。

模型为字符级自回归语言模型：

```text
输入 token 序列
-> 字符 embedding
-> 位置 embedding
-> 2 层 GRU
-> Linear 输出层
-> 预测下一个字符
```

统一训练三类任务：

| 任务 token | 含义 |
|---|---|
| `<TASK_FREE>` | 普通七绝建模 |
| `<TASK_CONT>` | 首句续写 |
| `<TASK_ACRO>` | 藏头诗生成 |

训练了三组 GRU 设置：

| 模型 | 说明 |
|---|---|
| baseline | 不使用句位标记，普通字符级 GRU |
| weighted | 不使用句位标记，但对藏头位置加大 loss 权重 |
| structured | 使用 `<L1><L2><L3><L4>` 句位标记，并在生成时使用结构约束 |

主要超参数：

| 参数 | 值 |
|---|---:|
| embedding dim | 256 |
| position dim | 32 |
| hidden size | 512 |
| GRU layers | 2 |
| dropout | 0.2 |
| batch size | 256 |
| epoch | 30 |
| learning rate | 0.0003 |
| weight decay | 0.01 |

## 4. 训练曲线

## 5. 自动评测结果

主实验使用 creative 采样设置进行三模型对比。

| 模型 | 生成方式 | Best Epoch | Test PPL | 格式合规率 | 藏头正确率 | Distinct-2 | 重复率 |
|---|---|---:|---:|---:|---:|---:|---:|
| baseline | raw | 7 | 83.690 | 1.000 | 0.000 | 0.9998 | 0.0002 |
| weighted | raw | 11 | 78.264 | 1.000 | 0.890 | 0.9993 | 0.0007 |
| structured | constrained | 7 | 51.185 | 1.000 | 1.000 | 0.9980 | 0.0020 |

结果说明：

| 结论 | 说明 |
|---|---|
| baseline 能学到七绝格式 | 格式率接近 1.000，但几乎不能稳定完成藏头 |
| weighted 明显改善藏头 | 通过提高藏头位置 loss 权重，藏头正确率提升到 0.890 |
| structured 整体最好 | 句位标记 + 结构约束使 Test PPL 最低，藏头正确率达到 1.000 |
